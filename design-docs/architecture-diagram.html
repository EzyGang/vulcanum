Vulcanum Agent Orchestrator Architecture

HTTP polling • Task manager integration • Tiered sandboxing • memfd secrets • API SKILL.md

Communication Model

  • • HTTP short polling (GET /poll, configurable 15s interval)
  • • Stateless — any LB can handle any request
  • • No persistent connections, no reconnection storms
  • • SSE available as optional push upgrade

Security Model

  • • Secrets NEVER in env vars (memfd injection)
  • • Config files generated in tmpfs, mounted read-only
  • • Sandbox: —unshare-net + egress filtering
  • • Output sanitization scans for leaked secrets

Agent-Native Control

  • • vulcanum-api SKILL.md shipped in CLI binary
  • • 1:1 API mapping — every endpoint callable from agent
  • • Works with Hermes, Claude Code, Codex CLI
  • • vulcanum install-skill --agent hermes

Isolation Tiers

  • • Tier 1 (Default): bubblewrap + nsjail (10-50ms)
  • • Tier 2 (Enhanced): Podman rootless (0.5-2s)
  • • Linux-only MVP — cross-platform deferred
  • • Both daemonless, both rootless-capable
