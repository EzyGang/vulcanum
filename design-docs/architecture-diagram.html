Vulcanum Architecture (MVP)

Kaneo → Main App → Worker (Firecracker + OpenCode) — thin metadata/trigger broker

Work Flow

  • • Main app polls Kaneo "todo" per enabled project
  • • INSERT work_run with ON CONFLICT DO NOTHING
  • • Worker short-polls cache flag → gets job_id
  • • Worker fetches full metadata (prompt, secrets, config)
  • • Boots Firecracker μVM, spawns OpenCode
  • • OpenCode does work, submits PR, exits
  • • Worker POSTs /result → Main app syncs Kaneo to "in review"

Isolation (Firecracker)

  • • Linux KVM required — Firecracker microVM per work item
  • • Pre-built rootfs with OpenCode + git + curl
  • • tmpfs workdir — ephemeral, destroyed on VM exit
  • • Network: egress to api.anthropic.com + github.com only
  • • CPU/memory limits enforced at VM level
  • • Secrets injected via env inside VM, never on host disk

MVP Boundaries

  • • Single-user, self-hosted, no auth gating on API
  • • Linux-only workers (Firecracker needs KVM)
  • • OpenCode as sole harness (no Claude Code / Codex)
  • • Kaneo as sole task source (no Linear / Jira)
  • • Plain HTTPS for secrets (agent-vault for v2)
  • • Main app is metadata broker — never validates code
