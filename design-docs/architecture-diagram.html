Vulcanum Architecture

Agent Orchestrator — Control Panel, Worker Sandbox, CLI

Isolation Strategy

  • • Sandbox via bubblewrap (Linux) or Podman + gVisor
  • • tmpfs filesystem — no persistent disk state
  • • Network disabled inside sandbox (no egress)
  • • cgroups v2 resource limits (CPU, memory, I/O)
  • • Namespace destruction on work completion (auto-cleanup)
  • • One agent harness per sandbox instance

Security Model

  • • mTLS between Worker and Main App (mutual auth)
  • • age (X25519) one-time secret wrapping at rest
  • • memfd in-memory secret injection (never touches disk)
  • • Magic-link email auth (token → verify) for users
  • • Secret Store age-encrypted at rest in PostgreSQL
  • • Worker behind NAT, outbound WebSocket only

Communication Protocols

  • • REST API: HTTPS on actix-web :8080
  • • WebSocket (WSS): persistent worker connections
  • • Worker polls Main App via outbound WebSocket (NAT-safe)
  • • mTLS secures all Worker ↔ Main App channels
  • • CLI ↔ Main App: REST + Magic-link auth flow
  • • Planned: Agent Server monitors via internal channels
